# Go Clean Architecture API

A production-grade, **Clean Architecture** (Hexagonal) REST API built with **Go** — designed to demonstrate enterprise-level engineering patterns, tooling, and best practices.

## Architecture

```
┌─────────────────────────────────────────────────────┐
│                    Delivery Layer                     │
│           HTTP Handlers (go-chi) + Middleware          │
├─────────────────────────────────────────────────────┤
│                   Use Case Layer                       │
│           Business Logic / Application Services        │
├─────────────────────────────────────────────────────┤
│                   Domain Layer                         │
│          Entities, Interfaces, Repository Specs        │
├─────────────────────────────────────────────────────┤
│                   Repository Layer                     │
│       sqlc-generated Type-Safe Go + pgx Connection Pool│
└─────────────────────────────────────────────────────┘
```

### Layer Isolation Rules

- **Domain**: Zero external dependencies. Pure Go types, interfaces, and business errors.
- **Use Case**: Only depends on Domain interfaces. Contains all business rules.
- **Repository**: Implements Domain interfaces. Uses sqlc + pgx for Postgres.
- **Delivery**: HTTP handlers, middleware, routing. Depends only on Use Case interfaces.

## Features

- **Clean Architecture** — strict layer separation; domain has zero dependencies
- **Type-Safe Database** — sqlc compiles SQL into type-safe Go with zero runtime queries
- **JWT Authentication** — access + refresh token flow with bcrypt password hashing
- **RBAC-ready** — middleware extracts user context for per-resource authorization
- **Rate Limiting** — Redis-backed sliding window rate limiter per IP
- **Structured Logging** — pretty-printed local logs, JSON in production (log/slog)
- **Graceful Shutdown** — handles SIGINT/SIGTERM with configurable timeout
- **Dockerized** — multi-stage Docker build + docker-compose with Postgres and Redis
- **Health Checks** — readiness endpoint for container orchestration
- **Comprehensive Tests** — mock-driven unit tests covering all business logic

## Quick Start

### Prerequisites

- Go 1.23+
- Docker & Docker Compose

### 1. Clone & Configure

```bash
git clone <repo-url>
cd go-clean-architecture
cp .env.example .env
```

Edit `.env` if needed (defaults are tuned for local Docker Compose).

### 2. Start Infrastructure

```bash
docker compose up -d postgres redis
```

### 3. Run Migrations

```bash
make migrate-up
```

### 4. Start the API

```bash
make run
```

The server starts at `http://localhost:8080`.

### Or: Run Everything with Docker

```bash
make docker-up      # Build and start all services
make docker-down    # Stop all services
```

## API Endpoints

### Health

```bash
curl http://localhost:8080/health
```

### Authentication

```bash
# Register
curl -X POST http://localhost:8080/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"name":"John Doe","email":"john@example.com","password":"secret123"}'

# Login
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"john@example.com","password":"secret123"}'
```

### Tasks with placeholders so u may edit however u want

```bash
# Set token
TOKEN="<your-access-token>"

# Create a task
curl -X POST http://localhost:8080/api/v1/tasks \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"title":"Buy groceries","description":"Milk, eggs, bread"}'

# List tasks
curl http://localhost:8080/api/v1/tasks \
  -H "Authorization: Bearer $TOKEN"

# Get task by ID
curl http://localhost:8080/api/v1/tasks/{taskID} \
  -H "Authorization: Bearer $TOKEN"

# Update task
curl -X PUT http://localhost:8080/api/v1/tasks/{taskID} \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"status":"completed"}'

# Delete task
curl -X DELETE http://localhost:8080/api/v1/tasks/{taskID} \
  -H "Authorization: Bearer $TOKEN"
```
